API keys

Request a personal API key.

An API key is required for all requests. In all code examples below substitute [YOUR_API_KEY] (including [¬†]) with your api key.

If an API key is not provided the following error is given:

{
    "errors": {
        "status": 401,
        "source": "/products",
        "title": "Valid API key",
        "detail": "No valid API key provided."
    }
}

id
article_number
name
description
specifiers
stock
location
price
api_key

GET /v2/products?api_key=[YOUR_API_KEY]

Code example:

                const response = await fetch(

To add a product:

POST /v2/products

Required parameters:

api_key

Parameters:

name
article_number
description
specifiers
stock
location
price

Code example:

                const product = {
    api_key: "[YOUR_API_KEY]",
    name: "Skruvdragare",
    article_number: "AAA111",
    price: 1000,
};

const response = await fetch("https://lager.emilfolino.se/v2/products", {
    body: JSON.stringify(product),
    headers: {
      'content-type': 'application/json'
    },
    method: 'POST'
});
const result = await response.json();

Result:

STATUS 201

    name: 'Skruvdragare',
    description: null,

  }
}

To update a product:

PUT /v2/products

Required parameters:

id
api_key

Optional parameters:

name
article_number
description
specifiers
stock
location
price

Code example:

    const product = {
    id: [ID_OF_ONE_OF_YOUR_PRODUCTS],
    api_key: "[YOUR_API_KEY]",
    name: "Skruvdragare",
    article_number: "AAA111",
    price: 1100,
};

const response = await fetch("https://lager.emilfolino.se/v2/products", {
    body: JSON.stringify(product),
    headers: {
      'content-type': 'application/json'
    },
    method: 'PUT'
});

Result:

HTTP Status Code 204 No Content

Possible error, besides errors returned from database:

{
    "errors": {
        "status": 400,
        "detail": "Required attribute product id (id) was not included in the request."
    }
}

Status

All orders have a status, it can take the following numeric values, corresponding to the descriptions. Orders have a default status value of 100.

100: "Ny"
200: "Packad"
400: "Skickad"
600: "Fakturerad"
800: "Retur"
900: "√Öterbetald"

api_key

name

Invoices and Login

An invoice has the following attributes:

id
order_id
total_price
creation_date
due_date
api_key

To use the following routes, you need a valid JSON Web Token (JWT) set in the HTTP-header.
The 'x-access-token' header should contain the JWT. 

GET /v2/invoices?api_key=[YOUR_API_KEY]
GET /v2/invoices/:invoice_id?api_key=[YOUR_API_KEY]
POST /v2/invoices

Register

POST /v2/auth/register

Required paramters:

api_key
email
password

Result:

{
    "data": {
        "message": "User successfully registered."
    }
}

Login

POST /v2/auth/login
api_key
email
password

Result:

{
    "data": {
        "type": "success",
        "message": "User logged in",
        "user": {
            "api_key": "...",
            "email": "unknown@example.com"
        },
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.ey..."
    }
}

N.B. The access token expires after 24 hours.

Initiate Payment Session

POST /v2/stripe/create-checkout-session

Required parameters:

return_url
items
api_key

Example:

                const items = [
    {
        price_data: {
            currency: 'sek',
            product_data: {
                name: 'Patch Notes',
            },
            unit_amount: 10000,
        },
        quantity: 1,
    },
    {
        price_data: {
            currency: 'sek',
            product_data: {
                name: 'Putrid Cathedral',
            },
            unit_amount: 10000,
        },
        quantity: 2,
    }
]

const data = {
    items: items,
    return_url: "https://github_user_name.github.io/webshop/order.html",
    api_key: "API_KEY",
}

const response = await fetch("https://lager.emilfolino.se/v2/stripe/create-checkout-session", {
    body: JSON.stringify(data),
    headers: {
    'content-type': 'application/json'
    },
    method: "POST",
})
const { clientSecret } = await response.json()

GET /v2/stripe/session-status

Required query parameters:

session_id
api_key

Example:

                const queryString = window.location.search;
const urlParams = new URLSearchParams(queryString);
const sessionId = urlParams.get('session_id');

if (sessionId) {
  const response = await fetch(`https://lager.emilfolino.se/v2/stripe/session-status?session_id=${sessionId}&api_key=e76d3a5f6e34ea474f5d303e08990b1e`);
  const session = await response.json();
}
